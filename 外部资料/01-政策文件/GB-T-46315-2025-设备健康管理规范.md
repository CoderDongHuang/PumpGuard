# GB/T 46315-2025《工业互联网平台 设备健康管理规范》

## 基本信息

| 项目 | 内容 |
|------|------|
| **标准号** | GB/T 46315-2025 |
| **发布日期** | 2025 年 10 月 5 日 |
| **归口单位** | SAC/TC573（两化融合）+ SAC/TC124（工业过程测量控制）联合归口 |
| **来源** | 国家标准全文公开系统 |

## 核心内容

### 四层框架

```
┌──────────────────────┐
│    工业应用层         │  ← 状态监测仪表板 / 健康评估报告 / 工单管理
├──────────────────────┤
│    工业 PaaS 层       │  ← AI 模型训练 / 健康评估引擎 / 故障诊断算法
├──────────────────────┤
│    工业边缘层         │  ← 协议转换 / 数据预处理 / 特征提取
├──────────────────────┤
│    工业设备层         │  ← 传感器 / PLC / 智能设备
└──────────────────────┘
```

### 健康管理全链条

涵盖四大核心活动：
1. **状态监测** — 实时采集运行参数
2. **健康状况评估** — 健康指数/退化趋势
3. **故障诊断** — 故障定位与根因分析
4. **维修维护** — 预测性维护计划与执行

融入 AI 等前沿技术，引导企业从"被动维修"向"主动服务"转型。

## 本方案对齐说明

| 国标层次 | 本方案对应 |
|---------|-----------|
| 工业设备层 | 数据接入层（新泵/老旧泵分层策略） |
| 工业边缘层 | 边缘网关（协议转换/清洗/特征提取） |
| 工业 PaaS 层 | 算法层（HI/RUL/根因分析） |
| 工业应用层 | 决策层 + 展示层（工单/看板/Bot） |
