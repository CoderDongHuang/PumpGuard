# Kaggle — Pump Sensor Data

> 极端类别不平衡场景下的罕见故障预测基准。

---

## 基本信息

| 项目 | 内容 |
|------|------|
| **名称** | Pump Sensor Data |
| **平台** | Kaggle（GitHub: dullibri/pump_sensor） |
| **设备** | 1 台泵 |
| **传感器数量** | **53 个** |
| **测量频率** | 每分钟 |
| **总时长** | 约 220,000 分钟（~153 天） |

## 极端不平衡

| 状态 | 时长 | 占比 |
|------|------|------|
| 正常运行 | ~205,000 分钟 | 99.997% |
| 故障状态 | **仅 7 分钟** | 0.003% |
| 恢复过程 | ~14,477 分钟 | 6.6% |

> 7 分钟故障 vs 22 万分钟总长 — 典型工业场景：故障极少，但后果严重。

## 已知 Notebook 探索

| 方法 | 效果 |
|------|------|
| 单传感器阈值 | 不可靠，大量误报 |
| 复合指标（统计偏差） | 误报问题未解决 |
| Logistic 回归（2 天预测窗口） | 5 次故障训练 / 2 次测试，捕获 1 次 |
| 建议下一步：LSTM / PCA / 偏差指标 | 未在公开 Notebook 中实施 |

## 本方案用途

- 罕见故障预测算法验证
- 极端类别不平衡处理策略测试
- 早期预警指标（EWI）设计参考
